Using the Xero Scouting System

Xero Scouting System Devices

There are three types of devices in the Xero Scouting System. These are the XeroCentral, the XeroScout, and the XeroCoach.
XeroCentral
This is the central device that contains all of the scouting data. This device creates and manages the event, including sychronization with all of the other types of devices.
XeroScout
This is the device a scouter uses. A XeroScout device can be used to scout another team or used to scout a match. Scouting assignments are given when a XeroScout device is first synchronized with the XeroCentral device. Subsequent synchronization operations moves the scouting data from the XeroScout device into the database on the XeroCentral device.
XeroCoach
The XeroCoach device contains a complete copy of the scouting data, like the XeroCentral device. However this data is (mostly) read only. The XeroCoach device is used by the team coach when reviewing strategy with other alliances about how to approach a match. Note, the XeroCoach device can connect to the Blue Alliance site to download match scoring data or zebra data. If this data is downloaded, it is transferred to the XereCentral device the next time these device synchronize.

Synchronization

Synchronization is the process of moving data from one device in the Xero Scouting System to another. Synchronization can occur over BlueTooth, TCP/IP, or USB. Before running synchronization, be sure to set the Team Number of the team using the Settings/Set Team Number menu item.
USB
USB synchronization happens over a special data transfer cable. This cable can be found on amazon via the link below. Once the two devices are connected via the USB cable, they can be syncronized via the Synchronize/USB Sync menu item.

TCP/IP
TCP synchronization does data transfer using the machines TCP/IP network. This could be WIFI or a hardwired Ethernet connection. There are two options here. Network Synchronization (Automatic) works on most environments. With automoatic synchronization, the XeroScout device is constantly looking for a beacon that is broadcast from the XeroCentral device advertizing that it is a central devices for the desired team. This beacon provides the IP address of the XeroCentral device and the XeroScouting device can connect.
If the XeroScout device does not see the XeroCentral beacon, the Network Synchronization (Direct) can be used. With direct synchronization, an IP address must be entered. This IP address can be found at the bottom status bar of the XeroCentral application.
Note, WIFI is illegal in the stands at an event. It interfers with the robots or the FTA systems on the field. If TCP/IP synchronization is to be used in the stands, it must be hardwired Ethernet. This generally requires a router of some type serving DHCP addresses (or statically assigned IP addresses for devices). This in turn requires some way to power a router in the stands. This gets to be complicated and prone to failures. The USB cable method is highly recommended for synchronization in the stands of an event.
BlueTooth
For two devices to synchronize using Bluetooth, the devices must first be paried. Under Windows 10 this is accomplished using the Settings/Devices window. Select "Add Bluetooth or other device" and follow the prompts. Once the two devices are paried, Bluetooth synchronization is started using the Synchronization/Bluetooth menu item.
Note, Bluetooth synchronization is the least reliable way to synchronize. In addition, Bluetooth operates on the 2.4 GHz frequency range which, while can interfer with the 2.4 Ghz WIFI at an event. Currently, there is no explicit rule against Bluetooth (that I have seen), but its used in less than ideal. It is highly recommended that the USB synchronization be used at events.

Create A New Event

A new event is created via the File/New Event menu item. Selecting this menu item will display a New Event Wizard. You must be connected to the internet to create a new event as the initial event data is downloaded from the Blue Alliance web site.

Select Event

On this page, select an event from the Blue Alliance database. Double click on the event to select it.

Select Forms

On this page, select the team scouting and the match scouting forms. These files are scouting forms defined in XeroScout JSON format. This format should be described in an accompying document.

Tablet Scouting Pool

On this page the set of tablets used for scouting is defined. In the Table Pool column you can add a new table by pressing the Insert key. You can delete a table by selecting the table and pressing the Delete key.
Once a set of tables is defined, drag tables from the Tablet Pool column to the Team Scout List column and the Match Scout List Column. You must have at least one tablet in the Team Scout List and six tablets in the Match Scout List to move forward.
Once the wizard is complete, the program reads the team list from the Blue Alliance web site. If the match schedule is available, it will also be downloaded, but this is not required at this stage.
At this point, the tablets assigned to team scouting are assigned to their respective teams. The Team Scouting Status window will show the team scouting tablet assignments.
Once all data is downloaded from the Blue Alliance site, you will be prompted to save the event. At this point, no more internet access is required unless you are getting Key Performance Indicators, Blue Alliance scoring data or Blue Alliance Zebra data.

Getting Key Performance Indicators From Blue Alliance

Key Performance Indicators is data about the performance of each robot in the current event, based on previous events in the same year. Getting this data involves determining every event a team has participated in for each team in the current event. Then all of the event data for each of these additional events is retreived. Finally all of this previou data is processed. For each entry in the scoring breakdown data from Blue Alliance, an average value is calculcated and added to the team data for each team.

Download Match Schedule

Once the match schedule is made available on the Blue Alliance, the match schedule can be downloaded. The match schedule is generally made available (at least for the PNW district events) the night of load-in as soon as all teams have checked in. It does no harm to try and download the match schedule and if the schedule it not available, a message is displayed.
At this point, the tablets assigned to match scouting are assigned to their respective matches. The Match Scouting Status window will show the match scouting tablet assignments.

Synchronizing with a Scouting Tablet

To start the scouting process, a XeroScout tablet must be sychronized with the XeroCentral device. Devices can be sychronized via USB, TCP/IP or Bluetooth. The first time a XeroScout devices is synchronized with the XeroCentral device, the XeroScout devices picks a table identity. This identity was established on the XeroCentral device when the event being scouted was created. Each tablet identity is assigned a set scouting assignments, either team scouting assignments or match scouting assignments. After the initial synchronization, these assignments are avaiable on the XeroScout device.

Windows

View Windows

The application consits of set of windows which can be selected from the left side view selector. Note that only the introduction page is visible until an FRC event data set is either loaded from disk or created.


Name Purpose
Introduction This window which provides an overview of the XeroScout scouting program.
Team Scouting Form A window that provides a view of the team (pit) scouting form. This is just to be sure what form actually gets downloaded to tablets. This form is not actually used to edit data.
Team Scouting Status A window that provides the status of team scouting. If the ScoutCentral program has team scouting data for a team, it will be indicated by a green 'Y'. If not, it is indicated by a red 'N'.
Team Scouting Data A dataset window to display the team scouting data collected to date, one team per row.
Match Scouting Form - Red A window that provides a view of the match scouting form when scouting the red alliance. This is just to be sure what form \actually gets downloaded to tablets. This form is not actually used to edit data.
Match Scouting Form - Blue A window that provides a view of the match scouting form when scouting the blue alliance. This is just to be sure what form actually gets downloaded to tablets. This form is not actually used to edit data.
Match Scouting Status A window that provides status information on match scouting. Data that is available is highlighed with a green 'Y'. Data that is not avaible is highlighed with a red 'N'. This status may be augmented with a 'Z' or 'B'. A 'Z' means that Blue Alliance Zebra data is available for the given match. A 'B' means that Blue Alliance scoring data is available for the given match
Match Scouting Data a dataset window to display the match data collected to date, one team and match per row. This will result in six total rows per match
Pre-Match Graphs A graph window that can be used to display data about the six teams in a specific match. The right click mouse will allow new graph panes to be created, or new variables or expressions to be added to or removed from existing graph panes
Single Team Summary A windows that displays a summary of a single team. Right clicking will display menus for adding values to be displayed in the team summary.
All Team Summary A dataset view that displays all scouting data, one row per team, across all teams. Numeric match scouting data is averaged across all matches played and displayed.
Alliance Graphs A graph window that can be used to display data about any arbitrary group of teams. The right click mouse will allow new graph panes to be created, or new variables or expressions to be added to or removed from existing graph panes
Pick List This windows shows the results of an external pick list analysis program. The program is assumed to be named analysisYYYY where YYYY is the current year. A picklist json file can be loaded via the file menu which maps the data in the scouting program, to output in a CSV file that will be used as input to the picklist program. This window displays two different HTML files generated by the external picklist program.
Custom Data A window that allow a general SQL query to be typed in and the results displayed in a dataset.
Zebra Introduction An introduction to the zebra data analysis methodology.
Zebra Data A graphical display of the zebra data either as a set of tracks, a heat map, or a replay of any selected matches. The data can be viewed per match or per robot. If viewed per robot, all matches for that robot will be displayed concurrently.
Zebra Region Editor A window to add, edit, or remove polygonial, rectangle, or circular regions on the field. The Zebra analysis can detect when a robot enters or leaves one of these regions.
Zebra Pattern Editor An editor for patterns of events to be detected and reported as robot activities.
Zebra Analysis This window displays the output of the zebra data analysis dislaying all detected sequences and their duration.
Change History A window that shows the history of changes to the scouting database.
Merge List This is for a future featuer. It provides a way to merge multiple scouting records for a given match or team.

Log Window


At the bottom of the main window, there is a log window that provides messages about interesting events that happen in the program. Keep a close eye on this window, especially if something goes wrong. Turning on the View/Debug menu item will cause more information to appear in the log window, especially during synchronization.
